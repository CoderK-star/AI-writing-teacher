# AI Writing Teacher プロジェクト計画

## 1. プロジェクトの目的

小説やラノベを書き始めたばかりの人に対して、
「何が悪いか」を突きつけるのではなく、
「次に何をすればよいか」を示せる創作教師AIを作る。

## 2. 想定ユーザー

- Web小説投稿初心者
- 小説を書きたいが、構成・描写・会話文の基礎に不安がある人
- 一気に完成度を上げるより、継続的に上達したい人

## 3. 初期PoCで絞る価値

- 書き方の講師
- プロット相談
- 短文改善提案

## 4. 初期PoCの成功条件

- 相談後に、ユーザーが次の執筆アクションを1つ以上決められる
- 回答が初心者に理解できる言葉で返る
- 回答が毎回「良い点 / 改善点 / 次の一歩」を含む
- 教材参照と学習者プロフィールが回答に反映される

## 5. 技術方針

### アプリ構成

- Webアプリ
- API経由で教師エージェントを呼び出す
- RAGと記憶基盤は差し替え可能な構成にする

### エージェント方針

- まずは単一エージェントで始める
- 将来は講師、プロット相談、添削、評価を別エージェントに分離可能にする

### RAG方針

- 最初は少量・高品質の教材だけを使う
- 検索結果をそのまま返すのではなく、学習者向けに言い換える

### 記憶方針

- 会話履歴と学習者プロフィールを分離する
- 保存するのは教育上意味がある特徴だけにする

## 6. ブレーンストーミング観点

- 初心者が最も詰まりやすいポイントはどこか
- 指導が厳しすぎる/抽象的すぎる瞬間はどこで起きるか
- 作品添削と書き方教育をどう切り分けるか
- 著作権リスクのある依頼にどう代替提案を返すか
- どの時点でマルチエージェント化する価値が出るか
