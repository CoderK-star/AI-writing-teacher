## 1. RAG がこのプロジェクトで果たす役割

AI Writing Teacher は創作指導エージェントです。LLM はそれ自体に汎用的な文章論の知識を持っていますが、そのまま使うと以下の問題が生じます。

| 問題 | RAG による解決 |
|------|--------------|
| 一般論的すぎる（ハウツー本の受け売り） | このプロジェクト専用に厳選した「指導しやすい粒度」の教材を注入 |
| ラノベ・Web小説の文脈が薄い | 対象読者（Web小説投稿初心者）に合ったトピックを優先して取得 |
| 毎回同じアドバイスを繰り返す | 会話文脈に近いチャンクを検索することで文脈に沿った指導が可能に |

RAGはシステム内で「参考教材コンテキスト」としてシステムプロンプトに埋め込まれ、エージェントはそれを言い換えながら指導します。教材本文をそのまま貼り付けることはしません。

## 2. アーキテクチャ概要

### 検索フロー

```
ユーザーメッセージ（直近3件のuserロール発言）
         │
         ▼
  OPENAI_API_KEY あり？
    ├─ Yes → ベクタ検索（text-embedding-3-small → cosine 類似度）
    │           ├─ 結果あり → 上位3チャンクを返す
    │           └─ 結果なし（インデックス空）→ キーワード検索にフォールバック
    └─ No  → キーワード検索（トークン一致スコア）
                └─ スコア上位3チャンクを返す
                        │
                        ▼
              formatRetrievedContext()
                        │
                        ▼
              システムプロンプトに「参考教材コンテキスト」として注入
                        │
                        ▼
              エージェント（lecture / plot / revision）が回答生成
```

### 使用コンポーネント

| ファイル | 役割 |
|---------|------|
| `lib/rag/corpus.ts` | 教材チャンクの静的定義（唯一の教材ソース） |
| `lib/rag/embedding.ts` | OpenAI Embeddings API でベクトル生成 |
| `lib/rag/vector-store.ts` | Vectra（ローカルファイル）によるベクタ検索 |
| `lib/rag/search.ts` | ベクタ検索 + キーワードフォールバックの統合エントリ |
| `scripts/seed-corpus.ts` | corpus.ts をベクタインデックスに一括登録するスクリプト |
| `.vectra/` | ベクタインデックスの保存先（自動生成、Git 管理外） |

### 設定値（`lib/agent/config.ts`）

| 設定 | デフォルト値 | 環境変数で上書き |
|------|------------|----------------|
| LLMモデル | `gpt-4.1-mini` | `OPENAI_MODEL` |
| 埋め込みモデル | `text-embedding-3-small` | `EMBEDDING_MODEL` |
| 最大取得チャンク数 | `3` | なし（コード修正が必要） |
| ベクタインデックス | `.vectra/` | なし |

## 3. 初期セットアップ

### 前提条件

- `OPENAI_API_KEY` が `.env.local` に設定されていること
- 依存パッケージのインストール済み（`npm install`）

```env
# .env.local
OPENAI_API_KEY=sk-xxxxxxxx
```

### ベクタインデックスの構築

```bash
npx tsx scripts/seed-corpus.ts
```

実行すると以下のような出力が得られます。

```
=== コーパスシード開始 ===
対象チャンク数: 6
インデックスを初期化しています...
埋め込みを生成してインデックスに登録しています...

✅ 完了: 6 チャンクをインデックスに登録しました。
インデックス保存先: C:\AI-writing-teacher\.vectra
```

> **注意**：教材（`corpus.ts`）を更新したら、必ず再度このスクリプトを実行して  
> インデックスを最新状態に保ってください。古いインデックスは自動的に上書きされます。

### OPENAI_API_KEY がない場合

キーを設定しなくてもアプリは動作します。その場合、ベクタ検索は使われず  
トークン一致によるキーワード検索が使われます。精度は落ちますが開発・テスト時に便利です。

## 4. チャンク追加ガイド

### チャンクの構造

`lib/rag/corpus.ts` に `KnowledgeChunk` 型のオブジェクトを追加します。

```typescript
{
  id: string;       // 一意な識別子（ケバブケース推奨）例: 'character-contrast'
  title: string;    // 1論点を端的に表すタイトル（20字以内が目安）
  topic: string;    // 大分類カテゴリ（後述の分類表を参照）
  content: string;  // 指導本文（100〜250字程度が理想）
  tags: string[];   // 検索補助キーワード（3〜6個推奨）
}
```

### トピック分類（`topic` フィールドの統一表記）

| topic 値 | 対象エージェントモード | 説明 |
|---------|-------------------|------|
| `導入` | lecture | 冒頭・つかみの技法 |
| `構成` | lecture | シーン設計・章構成 |
| `描写` | lecture | 地の文・五感描写 |
| `会話` | lecture | 台詞・会話文の技法 |
| `推敲` | revision | 文の磨き方・改善手順 |
| `プロット` | plot | 物語設計・転換点 |
| `キャラクター` | lecture / plot | キャラの個性・関係性 |
| `感情表現` | lecture / revision | 心理描写・感情の見せ方 |
| `世界観` | plot | 設定と描写のバランス |
| `テンポ` | lecture / revision | 文のリズム・速度調整 |

### 良いチャンクの例

```typescript
{
  id: 'emotion-show',
  title: '感情は反応と行動で見せる',
  topic: '感情表現',
  tags: ['感情', '描写', '心理', '反応'],
  content:
    '「悲しかった」と書く代わりに、その感情が現れる身体反応や行動を描くと読者が感情移入しやすくなる。視線が落ちる、食欲がなくなる、言葉が出ないなど。ただし重要でない感情まで全て描写すると文が重くなるため、場面の重要度で使い分ける。',
}
```

### 悪いチャンクの例（避けること）

```typescript
// ❌ 複数論点が混在している（1チャンクに収めすぎ）
{
  id: 'writing-tips',
  title: '良い小説を書くコツ',
  content: '冒頭を工夫し、会話を自然にし、描写を増やし、プロットを練り、推敲を丁寧にする。',
}

// ❌ 一般論すぎてWeb小説文脈に刺さらない
{
  id: 'general-writing',
  title: '文章を丁寧に書く',
  content: '読者のことを考えて分かりやすい文章を書きましょう。',
}

// ❌ 著作権のある書籍からの転載
{
  id: 'from-book',
  content: '「○○の法則」は△△著『◇◇』p.45 に記述されている通り…',
}
```

### 追加後の手順

1. `lib/rag/corpus.ts` にチャンクを追加
2. `npx tsx scripts/seed-corpus.ts` を実行してインデックスを再構築
3. アプリを起動して関連するトピックについて質問し、新しい教材が引用されることを確認する

## 7. コーパス品質チェックリスト

新しいチャンクを追加する前に、以下の項目を確認してください。

### 内容の品質

- [ ] **1チャンク1論点**になっているか（複数の技法が混在していないか）
- [ ] **タイトルが論点を端的に表している**か（「良い文章を書く」のような曖昧なタイトルではないか）
- [ ] **ターゲット（Web小説投稿初心者）の課題に直接刺さっているか**
- [ ] **自作の例文・解説のみで構成され、著作権を侵害していないか**
- [ ] **本文が 100〜250 字程度に収まっているか**（短すぎると情報量不足、長すぎると検索精度が落ちる）

### 検索性の品質

- [ ] **`topic` が定義済みの分類表から選ばれているか**（新規 topic を追加する場合は分類表も更新する）
- [ ] **`tags` が 3〜6 個で、本文に出てこないキーワードも含んでいるか**（別表現での検索に対応するため）
- [ ] **`id` がケバブケースで既存と重複していないか**

### 更新作業

- [ ] **corpus.ts の更新後に `npx tsx scripts/seed-corpus.ts` を実行したか**
- [ ] **追加したチャンクが実際の質問で取得されることを動作確認したか**

## 8. トラブルシューティング

### ベクタ検索の結果が返ってこない

**症状**：チャットで関連トピックを質問しても古い・的外れな回答が返る

**原因と対処**：

| 原因 | 対処 |
|------|------|
| `.vectra/` が存在しない | `npx tsx scripts/seed-corpus.ts` を実行 |
| `corpus.ts` を更新したがシードを再実行していない | 同上 |
| `OPENAI_API_KEY` が無効または未設定 | `.env.local` を確認。未設定でもキーワード検索で動作する |

### シードスクリプトがエラーで終了する

```
Error: OPENAI_API_KEY が設定されていません。.env.local を確認してください。
```

`.env.local` ファイルが存在するか、`OPENAI_API_KEY` が正しく設定されているかを確認してください。

```
Error: Request failed with status 429
```

OpenAI API のレートリミットに達しています。数秒待ってから再実行してください。  
チャンク数が多い場合は、スクリプトに `sleep` を挟む改修を検討してください。

### インデックスの件数と corpus.ts の件数が一致しない

```bash
# インデックスの件数を確認する（直接確認する方法）
npx tsx -e "
import { getIndexedCount } from './lib/rag/vector-store';
getIndexedCount().then(n => console.log('indexed:', n));
"
```

件数が一致しない場合は `.vectra/` ディレクトリを削除してからシードを再実行してください。

```bash
# Windows PowerShell
Remove-Item -Recurse -Force .vectra
npx tsx scripts/seed-corpus.ts
```

### キーワード検索モードで関連チャンクが取得されない

キーワード検索はトークンの一致に依存するため、質問文と教材本文で異なる表現が使われると  
取得精度が下がります。`tags` フィールドに同義語・別表現を追加することで改善できます。

```typescript
// 改善例: 「書き出し」「出だし」でも取得できるように
tags: ['冒頭', '導入', '引き', 'プロット', '書き出し', '出だし'],
```
